 1# Memoria Operacional - Ambiente pyenv + pyenv-virtualenv
 2
 3Data da verificacao: 2026-02-10
 4Branch: `refactor/fluxo-v2-documentado`
 5
 6## Objetivo
 7
 8Registrar, de forma curta e reutilizavel, como o ambiente Python deste projeto
 9deve ser configurado e usado para execucao do legado.
10
11## Estado confirmado no host (GeoServer)
12
13- `pyenv` instalado (`pyenv 2.6.20`).
14- plugin `pyenv-virtualenv` disponivel.
15- Ambientes do projeto existentes:
16  - `sismo-core-311` (Python 3.11.9)
17  - `sismo-rnc-379` (Python 3.7.9)
18
19## Arquivos canonicos de dependencias
20
21- `scripts/dev/requirements-core-pipeline.txt`
22  - stack do pipeline geral (pre, eventos, pos, mapas, relatorio).
23- `scripts/dev/requirements-rnc-legacy.txt`
24  - stack legado da RNC (inclui TensorFlow 2.8.0).
25
26Observacao: neste repositorio os nomes oficiais sao os acima (com sufixos
27`-pipeline` e `-legacy`).

47## Regra de execucao (ponto critico)
48
49Sem `PYENV_VERSION` (ou sem `.python-version`), o shell pode usar Python do
50sistema e gerar erros de dependencia faltante.
51
52Padrao adotado neste repositorio:
53
54- arquivo `.python-version` na raiz com `sismo-core-311`.
55- para etapa RNC (`-pr`), sobrescrever explicitamente com
56  `PYENV_VERSION=sismo-rnc-379`.
57
58Exemplos observados quando rodamos sem ambiente fixo:
59
60- `ModuleNotFoundError: shapely`
61- `ModuleNotFoundError: tqdm`
62- `ModuleNotFoundError: seaborn`
63- `ModuleNotFoundError: tensorflow`
64
65## Checklist rapido antes de testar
66
671. Confirmar ambiente ativo:
68
69```bash
70PYENV_VERSION=sismo-core-311 python -V
71PYENV_VERSION=sismo-rnc-379 python -V
72```
73
742. Rodar pipeline legado (etapas nao-RNC) no core:
75
76```bash
77PYENV_VERSION=sismo-core-311 bash .specs/codebase/fluxo_sismo.sh catalogo_jul_filtrado.csv -pe -e -po -m -r -t
78```
79
803. Rodar etapa RNC no ambiente legado:
81
82```bash
83PYENV_VERSION=sismo-rnc-379 bash .specs/codebase/fluxo_sismo.sh catalogo_jul_filtrado.csv -pr -t
84```
85
86## Decisao registrada
87
88- Mantemos dois ambientes separados por compatibilidade de dependencias.
89- O ambiente core nao deve receber TensorFlow legado.
90- A etapa `-pr` (RNC) deve ser executada no ambiente `sismo-rnc-379`.
91- No `fluxo_sismo.sh`, apenas o comando da predicao usa override de ambiente:
92
93```bash
94env PYENV_VERSION="$RNC_PYENV_VERSION" python fonte/rnc/run.py
95```
96
97Isso garante que `-pe`, `-e`, `-po`, `-m` e `-r` continuem no ambiente core.
