 1# Memoria: Dependabot x Ambiente RNC Legado
 2
 3Data da verificacao: 2026-02-11
 4
 5## Contexto
 6
 7No `origin/main` foram mesclados PRs Dependabot que alteraram `scripts/dev/requirements-rnc-legacy.txt`:
 8- PR #8: `protobuf 3.20.0 -> 5.29.6`
 9- PR #9: `tensorflow 2.8.0 -> 2.12.1`
10- PR #10: `ipython 7.28.0 -> 8.10.0`
11- PR #11: `numpy 1.21.3 -> 1.22.0`
12
13No branch `refactor/fluxo-v2-documentado`, o arquivo ainda permanece com os pins antigos (compatibilidade legada).
14
15## Ambiente legado atual
16
17`setup_pyenv_dual_envs.sh` define:
18- Python RNC: `3.7.9`
19- Virtualenv RNC: `sismo-rnc-379`
20
21## Diagnostico de impacto
22
23Os bumps do Dependabot quebram a reproducao do ambiente legado por incompatibilidade de versao:
24
251. `ipython==8.10.0` exige Python >= 3.8.
262. `numpy==1.22.0` exige Python >= 3.8.
273. `tensorflow==2.12.1` nao resolve em Python 3.7.
284. `tensorflow==2.12.1` conflita com `protobuf==5.29.6` (TF 2.12 exige `protobuf < 5`).
295. `tensorflow==2.12.1` conflita com `tensorflow-estimator==2.8.0` (TF 2.12 exige estimator >= 2.12).
306. Mesmo em Python 3.11, `tensorflow-io-gcs-filesystem==0.24.0` nao possui wheel compativel.
31
32## Conclusao operacional
33
34As atualizacoes de seguranca sao validas do ponto de vista de dependencia, mas nao podem ser aplicadas diretamente ao ambiente RNC legado sem estrategia de migracao de stack.
35
36Diretriz: separar trilhas de dependencia (legado x modernizada) antes de aplicar bump automatico no mesmo arquivo.
