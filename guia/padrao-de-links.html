Padrao De Links Da Documentacao¶

Objetivo¶

Definir regra unica de navegacao: toda mencao a arquivo, modulo, classe ou funcao deve apontar para uma pagina da documentacao, e nao para download.

Regra obrigatoria¶

  1. Arquivo citado -> link para pagina em Artefatos ou Anexos.

  2. Modulo, classe ou funcao Python citada -> link para API Reference.

  3. Nao usar :download: para arquivos de referencia da documentacao.

  4. Se a pagina do alvo ainda nao existir, criar a pagina antes de citar.

Exemplos validos¶

  • arquivo Bash: Artefato: fluxo_sismo.sh

  • requirements core: Artefato: requirements-core-pipeline.txt

  • anexo de matriz: Anexo: 0018-matriz-testes-fluxo-sismo-10.tsv

  • funcao de aquisicao: nucleo.fluxo_eventos.fluxo_eventos

  • funcao de pos-processamento: analise_dados.pos_processa.main

Observacao¶

Links para GitHub continuam permitidos como complemento dentro da pagina (documentacao <-> source), mas a navegacao primaria deve permanecer no site.
